# Les fondations de Cursor

<div class="grid grid-cols-3 gap-6 mt-4">

<div class="bg-blue-50 rounded-xl p-4">

### Un IDE pensé pour l'IA

<v-clicks>

- Un **éditeur à part entière**, pas un simple plug-in
- Interface optimisée pour coder avec des LLM
- L'IA aux **bons endroits** du workflow

</v-clicks>

</div>

<div class="bg-blue-50 rounded-xl p-4">

### Une base que les devs adorent

<v-clicks>

- À l'origine, un **fork de VS Code**
- Un cœur riche et éprouvé
- Import des **extensions & réglages** en un clic

</v-clicks>

</div>

<div class="bg-blue-50 rounded-xl p-4">

### Indexation de la codebase

<v-clicks>

- **Embeddings vectoriels** à l'échelle
- Réponses pertinentes, dans **votre style**
- Exploite dossiers & code comme contexte

</v-clicks>

</div>

</div>

<div class="mt-3 text-xs opacity-70 text-center">

**Anysphere**, l'éditeur de Cursor, a été fondé en **2022 par 4 étudiants du MIT** · aussi disponible en **plugin JetBrains** (Cursor ACP : IntelliJ, PyCharm, WebStorm…)

</div>

# Du CLI à l'IDE : Claude Code → Cursor

<div class="grid grid-cols-2 gap-8 mt-4">

<div>

### Ce que vous gardez

<v-clicks>

- Les **fondamentaux IA** : LLM, contexte, prompting
- La logique d'**agent** : plan, todos, tool-calling
- Les **règles** et bonnes pratiques (rules, MCP)
- Le réflexe « petit commit, tests, revue »

</v-clicks>

</div>

<div>

### Ce qui change avec Cursor

<v-clicks>

- L'IA **dans l'éditeur** : Tab, Cmd+K, Cmd+I
- Le **contexte visuel** : `@fichiers`, `@Docs`, indexation
- Le modèle **maison Composer 2.5**
- **Historique, canevas, automations** intégrés

</v-clicks>

</div>

</div>

<div class="mt-6 p-3 bg-blue-50 rounded-xl text-sm text-center">

L'idée du jour : capitaliser sur Claude Code et découvrir ce que Cursor apporte **nativement dans l'IDE**.

</div>
