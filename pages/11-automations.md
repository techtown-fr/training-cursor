# Automatiser avec des agents cloud

<div class="grid grid-cols-2 gap-8 mt-6">

<div>

### Le principe

<v-clicks>

- Des **cloud agents** lancés en arrière-plan
- Déclenchés sur **planning** ou sur **événement**
- Tournent toujours en **Max Mode**
- Facturés à l'usage des cloud agents

</v-clicks>

</div>

<div>

### Déclencheurs

<v-clicks>

- **Cron** / planning récurrent
- **GitHub / GitLab** : PR ouverte, mergée, CI finie…
- **Slack**, **webhooks**
- **Linear**, **Sentry**, **PagerDuty**

</v-clicks>

</div>

</div>

# Outils & cas d'usage

<div class="grid grid-cols-2 gap-8 mt-6">

<div>

### Outils disponibles

<v-clicks>

- **Ouvrir / commenter** une PR, demander des relecteurs
- **Envoyer / lire** sur Slack
- **MCP** : brancher des outils externes
- **Mémoires** : notes persistantes entre exécutions

</v-clicks>

</div>

<div>

### Exemples concrets

<v-clicks>

- **Revue** des derniers commits d'une PR
- **Triage** des bugs signalés dans Slack
- Investigation auto d'une **erreur Sentry**
- **Digest quotidien** des changements du repo

</v-clicks>

</div>

</div>

<div class="mt-6 p-3 bg-blue-50 rounded-xl text-sm text-center">

On crée une automation depuis l'**Agents Window**, sur **cursor.com/automations** ou via un **template du Marketplace**.

</div>
